## Project Purpose

Converts scientific research article PDFs into LLM-readable Markdown. Figures and images are replaced with detailed textual descriptions generated by a vision model (Gemini Flash), so the final output is fully understandable by text-only LLMs.

## Running

```bash
pip install -r requirements.txt
export GEMINI_API_KEY="your-key"
python process.py input.pdf                # outputs input.md
python process.py input.pdf -o output.md   # custom output path
python process.py input.pdf --model gemini-2.5-flash  # override model
```

## Architecture

Single-file pipeline in `process.py` with three stages:

1. **PDF → Markdown + images**: Docling parses the PDF into structured Markdown with `<!-- image -->` placeholders, and extracts figure images as PIL objects
2. **Context building**: For each placeholder, extracts the enclosing section text from the Markdown and scans the full document for cross-references ("Figure N" mentions in other sections)
3. **Image interpretation**: Each image + its context is sent to Gemini Flash, which returns a textual description. Placeholders are replaced with `> **[Interpreted Image N]**` blockquotes. Failed interpretations are marked with `[WARNING: ...]` so downstream consumers can detect issues

Key design constraint: the script is a clean CLI tool (file in → file out) so it can be trivially wrapped as an MCP tool for the avisomo-agent-collective agent system.

## Dependencies

- `docling` — IBM's PDF-to-Markdown converter (layout analysis, table extraction, figure extraction)
- `google-genai` — Google GenAI SDK for Gemini Flash vision model calls

## Environment Variables

- `GEMINI_API_KEY` — required, Google AI Studio API key
