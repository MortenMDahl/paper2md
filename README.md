# paper2md

Convert scientific research PDFs into LLM-readable Markdown. Images and figures are extracted and replaced with detailed textual descriptions generated by a vision model, so the final output is fully understandable by text-only LLMs.

## How it works

1. **PDF parsing** — [Docling](https://github.com/docling-project/docling) (IBM) converts the PDF into structured Markdown, preserving headings, tables, and reading order. Figures are extracted as images.
2. **Context gathering** — For each image, the script collects the full enclosing section and scans the rest of the paper for cross-references (e.g. "as shown in Figure 3") to build accurate context.
3. **Image interpretation** — Each extracted image + its context is sent to Gemini Flash, which returns a detailed textual description covering data, trends, axes, statistical indicators, and scientific conclusions.
4. **Output** — The image placeholders are replaced with clearly-marked `[Interpreted Image N]` blockquotes, producing a single Markdown file that captures the full content of the paper.

## Setup

```bash
pip install -r requirements.txt
```

Copy `.env.example` to `.env` and add your [Google AI Studio](https://aistudio.google.com/) API key:

```
GEMINI_API_KEY=your-api-key-here
```

## Usage

```bash
python process.py paper.pdf                           # outputs paper.md
python process.py paper.pdf -o output.md              # custom output path
python process.py paper.pdf --model gemini-2.5-flash  # override model
```

## Output format

Original images are replaced with interpreted blockquotes:

```markdown
> **[Interpreted Image 1]**
> This image presents a bar chart comparing silicon uptake (mg/kg dry weight)
> across three treatment groups (Control, 50ppm Si, 100ppm Si). The y-axis
> ranges from 0 to 450 mg/kg. Treatment at 100ppm shows a 40% increase over
> control (p < 0.05), with error bars indicating standard deviation across
> n=6 replicates. This supports the paper's conclusion that silicon
> supplementation enhances uptake in a dose-dependent manner.
```

Images that fail interpretation are marked with a `[WARNING: ...]` tag in the output so downstream consumers know which images weren't processed.

## Requirements

- Python 3.10+
- [Docling](https://github.com/docling-project/docling) — PDF-to-Markdown conversion
- [google-genai](https://github.com/googleapis/python-genai) — Gemini API SDK
- [python-dotenv](https://github.com/theskumar/python-dotenv) — `.env` file loading
